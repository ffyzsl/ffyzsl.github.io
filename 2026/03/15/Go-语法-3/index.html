1. 变量逃逸分析 (Escape Analysis)

什么是逃逸分析？

在C/C++中，开发者需要手动管理内存（分配和释放）。而在Go中，编译器非常聪明，它会自动决定一个变量是分配在栈（Stack）上，还是分配在堆（Heap）上。这个决定的过程，就叫逃逸分析。

  • **栈 (Stack)**：分配快，释放快，函数结束自动回收。
  • **堆 (Heap)**：分配慢，需要垃圾回收器（GC）来回收，会带来额外的CPU消耗。

面试核心考点：什么时候会发生逃逸？

如果编译器发现一个变量在函数结束后还需要被外部访问，它就会把这个变量“踢”到堆上，这就是逃逸。

常见逃逸场景：

  1. 函数返回局部变量的指针：这是Go特有的魔法。在C语言中这是绝对禁止的（悬挂指针），但在Go中，编译器发现你返回了指针，就会把这个变量移到堆上。
  2. 变量过大：栈空间有限，太大的对象直接扔堆里。
  3. interface{} 动态类型：比如 fmt.Println(a)，因为 Println 接收的是 interface{}，编译器很难确定具体类型大小，往往会逃逸。

如何查看逃逸？

面试官可能会问：“你怎么知道变量逃逸了？”
命令： go build -gcflags "-m" main.go

2. 变量的生命周期

变量活多久，取决于它在哪里。

  1. 全局变量：程序启动时创建，程序结束时销毁。
  2. 局部变量（未逃逸）：函数调用时创建，函数结束时立刻销毁（栈内存）。
  3. 局部变量（已逃逸）：函数调用时创建，但因为它在堆上，直到没有任何人引用它时，才会被垃圾回收器（GC）清理。

面试总结：Go语言的变量生命周期不由作用域唯一决定，还取决于逃逸分析的结果。

3. 常量 (Constants)

常量就是在编译期间就确定的值，程序运行过程中无法改变。

关键字

const

特性

  • 常量不使用也不会报错（变量不使用会报错）。
  • 常量只能是基础类型（布尔、数值、字符串）。

面试核心考点：iota (常量计数器)

iota 是Go语言的一个黑科技，专门用于批量定义常量。

  1. const 出现时，iota 重置为 0。
  2. 每新增一行常量声明，iota 自动加 1。

5. 类型别名 (Type Alias) vs 类型定义

这是个容易混淆的概念，注意看等号 =。

1. 类型定义 (Type Definition)

type NewInt int

  • NewInt 是一个全新的类型。
  • 它和 int 不能直接赋值，必须强转。
  • 它可以拥有自己的方法。

2. 类型别名 (Type Alias)

type MyInt = int (注意这个等号)

  • MyInt 只是 int 的一个绰号。
  • 它们本质上是同一个类型，可以直接赋值。
  • Go源码中 byte 就是 uint8 的别名，rune 就是 int32 的别名。

6. 注释定义

Go的注释不仅是给人看的，有时候也是给编译器看的。

  1. 单行注释：//
  2. 多行注释：/* ... */
  3. 特殊指令注释 (进阶)：
    • //go:build linux：告诉编译器只在Linux下编译该文件。
    • //go:embed：打包静态资源。

面试提示：好的Go代码，导出的函数（首字母大写）必须要有注释，且注释应该以函数名开头。

7. 关键字和标识符

关键字 (Keywords)

Go只有25个关键字，非常简洁。
break, default, func, interface, select, case, defer, go, map, struct, chan, else, goto, package, switch, const, fallthrough, if, range, type, continue, for, import, return, var

标识符 (Identifiers)

就是你给变量、函数起的名字。

  • 只能由字母、数字、下划线组成。
  • 不能以数字开头。
  • _ (下划线) 是特殊标识符（空标识符），用于丢弃值。

8. 运算符的优先级

面试中不会让你背表格，但有几个常识要懂：

  1. 最高级：* / % << >> &
  2. 次级：+ - | ^
  3. 比较：== != < <=
  4. 逻辑：&& 高于 ||
  5. 赋值：= += 等最低。

老鸟建议：记不住优先级没关系，**多用括号 ()**！代码可读性比炫技更重要。

9. 字符串和数值类型的相互转换 (重点)

Go不像动态语言那样会自动转换，必须使用标准库 **strconv**。

场景一：String 转 Int

使用 strconv.Atoi (ASCII to Integer)

场景二：Int 转 String

使用 strconv.Itoa (Integer to ASCII)

场景三：Parse 系列 (更灵活)

strconv.ParseInt, strconv.ParseFloat, strconv.ParseBool

☠️ 面试超级大坑：string() 强转

千万不要用 string() 直接转 int！

总结一下

  1. 逃逸分析：理解栈和堆的区别，知道 go build -gcflags "-m"。
  2. 常量：掌握 iota 的自增和跳过机制。
  3. 枚举：Go没枚举，用 const + type 模拟。
  4. 别名：= 是别名（完全一样），没 = 是新类型（需要强转）。
  5. 转换：数字转字符串用 strconv，别乱用 string(int)。
