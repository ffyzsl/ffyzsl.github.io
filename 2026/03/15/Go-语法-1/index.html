Go 语法(1)

发表于 2026-03-15

1. 变量的声明 (Declaration)

在Go语言中，变量声明不仅仅是起个名字，它还涉及到内存分配和零值（Zero Value）的概念。Go是静态强类型语言，这意味着每个变量都有固定的类型。

标准声明格式

使用 var 关键字：var 变量名 变量类型

面试核心考点：零值机制

如果你声明了一个变量但没有给它赋值，Go不会像C语言那样给你一个随机的垃圾值，而是会给它一个默认的零值。

  • int -> 0
  • float -> 0.0
  • bool -> false
  • string -> "" (空字符串)
  • 指针、切片、Map、通道、接口 -> nil

2. 变量的初始化 (Initialization)

初始化就是在声明的同时给它一个值。Go提供了三种主要方式，面试中要特别注意第三种的限制。

方式一：标准格式

var name type = value

方式二：类型推导 (Type Inference)

编译器很聪明，可以根据右边的值自动推断左边的类型。
var name = value

方式三：短变量声明 (Short Variable Declaration) (重点)

name := value
这是Go最常用的写法，简洁高效。

面试核心考点：:= 的坑

  1. 只能在函数内部使用：你不能在全局（函数外部）使用 :=，全局变量必须用 var 开头。
  2. 左边必须有新变量：:= 左边的变量中，至少要有一个是新声明的，否则编译报错。

3. 多个变量同时赋值

Go语言支持并行赋值，这让代码非常优雅。

语法

var a, b = 1, 2 或 a, b := 1, 2

面试核心考点：交换变量

在其他语言（如Java/C）中，交换两个变量通常需要一个中间变量 temp。在Go中，你可以直接利用多重赋值来实现，这也是面试中展示你“Go味儿”正不正宗的一个小细节。

4. 匿名变量 (Anonymous Variable)

在使用Go函数时，经常会遇到函数返回多个值（比如 result, error），但有时候我们只需要其中一个值。如果你声明了变量却不使用，Go编译器会报错（Unused variable）。

这时候，下划线 _ 就登场了。

概念

_ 被称为匿名变量，也叫哑元变量。

面试核心考点

  1. 不占用内存：匿名变量不占用命名空间，也不会分配内存。
  2. 可重复声明：你可以在代码里多次使用 _。
  3. 丢弃数据：任何赋给 _ 的值都会被直接丢弃。

代码演示

5. 变量的作用域 (Scope)

变量在哪里能被访问，在哪里不能，这就是作用域。

分类

  1. 局部变量：在函数体内或代码块 {} 内声明。只在当前块有效。
  2. 全局变量：在函数体外声明。在当前包（package）内都有效。

面试核心考点：变量遮蔽 (Variable Shadowing)

如果局部变量和全局变量同名，局部变量会覆盖（遮蔽）全局变量。这是很多Bug的根源。

进阶考点：首字母大小写

  • 首字母大写（如 Name）：公开的（Public），其他包可以访问。
  • 首字母小写（如 name）：私有的（Private），只有当前包内可以访问。

代码演示

总结一下

  1. 声明：记得 var，不赋值就是零值。
  2. 初始化：:= 最常用，但别在全局用。
  3. 多重赋值：a, b = b, a 交换变量很帅。
  4. 匿名变量：_ 是个垃圾桶，不用的返回值扔进去。
  5. 作用域：就近原则，局部优先于全局；首字母大写走遍天下，小写只能窝在家里。
