Go 语法(2)

发表于 2026-03-15

1. 整型 (Integer)

分类

  1. 有符号整型：int8, int16, int32, int64
  2. 无符号整型：uint8, uint16, uint32, uint64

面试核心考点：int 和 int64 是一样的吗？

答案：不一样。

  • int / uint：这是平台相关的类型。
    • 在32位系统上，int 约等于 int32。
    • 在64位系统上，int 约等于 int64。
  • 坑点：虽然在64位机器上 int 和 int64 都是64位，但它们是不同的类型。你不能把 int64 的变量直接赋值给 int 类型的变量，必须强转！

2. 浮点类型 (Float)

Go只有 float32 和 float64，没有 double 这个关键字（float64 就相当于 double）。

面试核心考点：精度问题

  1. 默认推导：f := 3.14，Go编译器默认推导为 **float64**，因为它的精度更高。
  2. 精度丢失：这是计算机科学的通病，不是Go特有的，但在涉及金钱计算时面试官会问。

3. 复数 (Complex)

这个在日常Web开发用得少，但在算法和科学计算中会用到。

  • complex64：实部和虚部都是 float32。
  • complex128：实部和虚部都是 float64（默认）。

代码演示

4. Bool 类型

Go的 bool 类型非常严格。

面试核心考点：不能强转

在 C/C++ 或 Python 中，非0即真，0即假。但在 Go 中：

  1. bool 只能是 true 或 false。
  2. bool 不能和 int 相互转换。你不能写 if 1 { ... }，必须写 if 1 == 1 { ... }。

5. 字符串 (String)

Go的字符串是面试中的重灾区。

特性

  1. UTF-8编码：Go天生支持UTF-8，处理中文非常方便。
  2. 不可变性：字符串一旦定义，其内容不能修改（类似Java）。

定义方式

  • **双引号 ""**：识别转义字符（如 \n 换行）。
  • **反引号 `**：原生字符串，不识别转义，所见即所得（常用于写SQL、JSON、多行文本）。

6. 字符类型：Byte 和 Rune (重点)

这是Go语言最独特的地方，也是面试必考。

区别

Go语言中没有 char 类型，而是提供了两种字符类型：

  1. byte (即 uint8)：
    • 代表一个ASCII码字符。
    • 占用 1个字节。
  2. rune (即 int32)：
    • 代表一个UTF-8字符（Unicode码点）。
    • 占用 4个字节。
    • 处理中文必须用 rune。

面试核心考点：len() 到底算的是什么？

  • len(str) 计算的是字节数。
  • 如果要计算字符数量（比如”你好”是2个字），需要转成 []rune 或使用 utf8.RuneCountInString。

7. 数据类型转换

Go语言没有隐式类型转换（Implicit Conversion），所有的转换必须显式进行。

语法

T(v)：将值 v 转换为类型 T。

面试核心考点：截断与溢出

将大范围类型转为小范围类型（如 float 转 int，int64 转 int8）时，会发生精度丢失或数据截断。

8. 指针 (Pointer)

Go保留了指针，但做出了巨大的简化。

核心概念

  • & (取地址符)：获取变量的内存地址。
  • * (取值符)：根据地址获取对应的值。

面试核心考点：Go指针 vs C指针

Go的指针不支持指针运算。你不能像C语言那样写 ptr++ 来移动指针位置。这大大提高了安全性，避免了非法内存访问。

代码演示

总结一下

  1. 整型：int 在不同机器上长度不同，别混用 int 和 int64。
  2. 浮点：默认 float64，别用 == 判等。
  3. Bool：它就是它，不是 0 或 1。
  4. 字符串：不可变，底层是字节数组。
  5. Byte vs Rune：这是重点！处理ASCII用 byte，处理中文/特殊字符一定要用 rune。
  6. 转换：必须显式强转，注意精度丢失。
  7. 指针：只能看和改值，不能乱动位置（无运算）。
