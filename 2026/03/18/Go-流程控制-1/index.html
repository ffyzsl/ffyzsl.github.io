Go 流程控制(1)

发表于 2026-03-18

1. 分支结构 (if / else)

Go的 if 看起来和C/Java很像，但有两个关键区别：不需要括号，以及支持初始化语句。

💡 面试考点：变量的作用域

面试官经常会看你写的代码够不够“Go味”。在Go中，我们非常习惯在 if 的条件判断前执行一个简单的语句（通常是获取返回值和错误），这样声明的变量作用域仅限于这个if-else块，能有效避免变量污染。

2. 循环结构 (for)

Go语言中没有 while 和 do-while！ 所有的循环操作，全部由 for 关键字包揽。

💡 面试考点：如何用 for 实现各种循环？

你需要熟练掌握 for 的几种变体。

3. 键值循环 (for range) —— 🚨 面试高频重灾区！

for range 是Go遍历数组、切片(slice)、字符串、字典(map)和通道(channel)的神器。

💡 面试大坑：range 的值拷贝问题

这是面试最爱考的题目，没有之一！ 面试官会给你一段代码，问你遍历修改切片里的结构体属性，最后有没有修改成功。

记住核心结论：for index, value := range slice 中的 value 是原始数据的一个拷贝！直接修改 value 不会影响原切片！

4. switch 语句

Go的 switch 比其他语言聪明得多。它可以判断任意类型（不仅限于常量或整数），而且默认自带 break。

💡 面试考点：fallthrough 和 无条件 switch

在C/Java中，如果你忘了写 break，代码会继续往下执行（case穿透）。Go反其道而行之，默认不穿透。如果你非要穿透，得显式使用 fallthrough 关键字。

5. goto 语句

goto 可以无条件地转移到程序中指定的标签（Label）。

💡 面试考点：什么时候该用 goto？

面试官如果问你对 goto 的看法，千万不要一棍子打死说绝对不能用。
标准回答：“在Go中应尽量避免使用 goto，因为它会破坏代码的可读性。但在某些特定场景下，比如跳出深层嵌套循环或者集中处理错误清理工作时，使用 goto 反而能让代码更简洁（Go源码中就有很多这样的用法）。”

6. break 与 continue

这两个老朋友的作用和别的语言一样：

  • break：彻底结束当前循环（或 switch）。
  • continue：跳过本次循环剩下的代码，直接进入下一次循环。

💡 面试高级考点：配合 Label (标签) 使用

在多层嵌套循环中，普通的 break 只能跳出最内层循环。Go 允许 break 和 continue 配合标签使用，精准控制跳出哪一层。

总结给新手的面试建议：

  1. 遇到 for range 题目，立刻警惕值拷贝陷阱。
  2. 遇到 switch 题目，注意找有没有 fallthrough。
  3. 遇到多层循环跳出，记得提一嘴 break Label 或者 goto。
