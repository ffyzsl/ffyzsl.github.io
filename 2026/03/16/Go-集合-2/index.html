1. Range

range 是Go语言中用于遍历数组、切片、Map、字符串和通道（Channel）的关键字。看似简单，但这里藏着Go语言最经典的面试坑。

核心机制

  • 值拷贝：for i, v := range slice 中，这个 v 是切片元素的副本。
  • 地址复用（Go 1.22之前）：在Go 1.22版本之前，v 的内存地址在整个循环中是不变的。虽然Go 1.22修复了闭包捕获的问题，但值拷贝的特性依然存在。

面试官爱问：
Q: 在 range 遍历切片时，我向切片 append 元素，循环会死循环吗？
A: 不会。因为 range 在开始遍历前，会先获取切片的长度 len，循环次数在开始时就已经决定了。

3. Map (映射/哈希表)

Map是Go语言中最强大的键值对结构。它是引用类型。

核心规则（背下来！）

  1. 无序性：Map的遍历顺序是随机的！不要依赖遍历顺序。
  2. Key的限制：Key必须是可比较的类型（如int, string, bool, 指针, 结构体）。切片、函数、Map不能做Key。
  3. 零值陷阱：声明但未初始化的Map是 nil。读取 nil map 返回零值（安全），但向 nil map 写入会导致 panic（崩溃）。

5. Map 的删除和清空

删除单个键值对

使用内置函数 delete(map, key)。

  • 如果 key 存在，删除之。
  • 如果 key 不存在，什么都不发生，也不会报错。

清空 Map

  • Go 1.21 之前：没有直接清空的函数。通常做法是 m = make(map[...]) 创建一个新的，旧的交给GC回收；或者遍历删除（效率低）。
  • Go 1.21 之后：新增了内置函数 clear(m)，效率极高。

6. Map 的多键索引 (Multi-key)

Go不支持 map[k1, k2]val 这种语法。如果你需要用两个ID来确定一个值，怎么办？

方案一：嵌套 Map (不推荐)

map[string]map[string]int。初始化麻烦，每一层都要检查 nil。

方案二：结构体作为 Key (推荐)

这是Go语言的一个妙招。因为结构体只要字段可比较，它本身就是可比较的。

7. sync.Map (并发安全 Map)

这是面试的高频考点。

为什么需要它？

标准的 map 不是并发安全的！
如果在多个 goroutine 中，一个在读，一个在写（或者两个都在写），程序会直接 Panic 崩溃（fatal error: concurrent map writes）。

解决方案

  1. 加锁：使用 sync.RWMutex 保护标准 map（最常用）。
  2. sync.Map：Go语言自带的并发安全 map。

sync.Map 特点

  • 开箱即用：不用 make，直接 var m sync.Map。
  • 性能场景：适合读多写少的场景。如果写操作非常频繁，性能不如“标准Map + 锁”。
  • 操作方式：不能用 m[k]，必须用特定方法。

代码演示

面试官总结：

  • 普通 Map 读写不安全，并发写必死。
  • 解决并发：首选 Mutex + map（通用性好），特定场景（读多写少、键值对不相交）用 sync.Map。
  • Map 的 Key 不能是切片。
  • Map 遍历顺序是随机的。
